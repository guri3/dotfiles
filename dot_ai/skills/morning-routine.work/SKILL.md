# morning-routine.work

## 目的

朝の挨拶をトリガーに、始業時の定常業務（昨日の振り返り・業界リサーチ・今日のスケジュール確認）を 1 度のリクエストで一括実行するスキルである。
人間が「おはよう」と言うだけで、その日のキャッチアップに必要な情報がまとまった状態でチャットに届く状態を目指す。

## 使うタイミング

- 「おはよう」「おはようございます」「good morning」と挨拶されたとき
- 「朝のルーティン」「今日も始めよう」「始業」など一日の開始を示す言葉が出たとき
- 上記が単なる挨拶ではなく、明らかに作業開始の文脈である場合に積極的に発動する

挨拶への単なる返事として処理せず、必ず定常業務の自動実行に切り替えること。

## 手順

`daily-review.work` / `poikatsu-research.work` 等の単機能スキルと組み合わせる前提のラッパーである。
個別 skill が利用可能ならそれらを呼び、無ければ inline で軽量版を実行する。

1. 昨日の振り返り
    - `daily-review.work` 相当を実行する
    - 自分の GitHub 活動（PR / commit / コメント）を取得する
        - `gh search prs --author @me --updated ">=YYYY-MM-DD"` 形式で昨日以降の PR を検索する
        - 必要なら `gh search issues --author @me --updated ">=YYYY-MM-DD"` でコメント付き Issue も拾う
    - Slack MCP が使えれば、自分の昨日の post を `slack:slack-search` 等で取得する
    - 取得結果から「何をしたか／何が残っているか」を簡潔にサマリする
2. 業界リサーチ
    - `poikatsu-research.work` 相当を実行する
    - チームのドメイン（ポイ活サービス）に関する直近のニュースを WebSearch で収集する
    - 大手競合（ポイントサイト各社）の動きを軽く眺め、目立つ話題があればピックアップする
    - 量より要点。3〜5 件の見出し＋一言コメント程度に圧縮する
3. 今日のスケジュール確認（任意）
    - Google Calendar MCP が利用可能であれば `mcp__claude_ai_Google_Calendar__list_events` で今日の予定を取得する
    - 利用不可ならこのステップは省略してよい
4. すべてを 1 つのチャット出力にまとめる
    - 見出し構成例:
        - 「昨日の振り返り」
        - 「業界リサーチ」
        - 「今日の予定」（取得できた場合）
    - 末尾に「今日の優先タスクは？」など軽い問いを添えて、ユーザーが次の動作に入りやすいようにする

## 注意

- 単発の「おはよう」だけで重い処理が走るので、ユーザーが既に作業中の流れに割り込まないか軽く判断する。明らかに別作業の途中ならスキップ提案してよい
- WebSearch / Slack MCP / Google Calendar MCP のうち、利用できないものは黙ってスキップし、出力にも理由を一言だけ添える
- 業界リサーチの出典は必ず URL を併記する。要約の根拠を辿れない情報は載せない
- 個別 skill（`daily-review.work` / `poikatsu-research.work`）の責務をこのスキル側で勝手に拡張しない。本スキルはあくまでラッパーである
- 出力は 1 つのチャットに集約する。複数メッセージに分割しない
